# Intruder Snapshot System (防偷窥/抓拍系统) 设计架构

## 核心目标
在“极速防范偷窥”与“控制误报率”之间取得完美的平衡。避免因合法用户佩戴口罩、光线不佳等正常识别失败导致大量无效抓拍（塞满硬盘），同时确保真正的入侵者被精准记录。

## 1. 双重阈值机制 (Dual Thresholds)
将人脸匹配得分划分为三个清晰的区间，彻底剥离“识别失败”与“入侵”的概念：
- **匹配放行阈值 (Match Threshold)**: 得分高于此值，解锁成功。
- **正常误差区间 (Normal Error Zone)**: 得分介于两个阈值之间。判定为可能是合法用户但条件不佳（如戴口罩、打哈欠、光线差）。不解锁，**坚决不抓拍**。
- **入侵者阈值 (Intruder Threshold)**: 得分低于此极低阈值。系统认定当前人脸“绝对不是本人”。进入抓拍判定逻辑。

## 2. 场景分离验证 (Scenario-based Verification)
根据系统唤醒方式的不同，采取严格区分的抓拍判定策略。

### A. 主动唤醒 (Active Wakeup)
- **触发条件**: 通过键盘敲击、鼠标点击等动作主动唤醒屏幕并触发识别。
- **抓拍逻辑**: 只要人脸得分低于“入侵者阈值”，**立即抓拍**！
- **设计依据**: 既然有物理上的主动交互动作，且人脸特征极度不匹配，说明有非授权人员正在尝试操作电脑。

### B. 静默扫描 (Passive / Silent Scanning)
- **触发条件**: 系统在锁屏后台持续静默扫描（防偷窥模式）。
- **抓拍逻辑**: 必须**同时满足**以下三个严苛条件才进行抓拍：
  1. **非本人**: 人脸得分低于“入侵者阈值”。
  2. **大脸占比 (Face Proportion)**: 人脸在画面中占据较大比例。证明对方距离屏幕足够近，排除了远距离走过的人（如保洁阿姨路过）。
  3. **盯着屏幕 (Head Pose)**: 人脸的 Yaw (偏航角) 和 Pitch (俯仰角) 几乎为 0。证明对方视线正对着屏幕，有明确的“偷窥”意图。
- **设计依据**: 叠加上下文状态，精准狙击真正的偷窥者，零误报。

## 模块集成计划
本系统将作为一个独立的模块集成到 WinFaceUnlock 架构中：
1. **优先保障主线**: 待 Rust workspace、Credential Store、IPC 和 Windows Service 主干链路跑通后引入。
2. **Face Engine 依赖**: 要求底层人脸识别引擎不仅输出匹配度分数，还需稳定输出 Bounding Box (用于计算大脸占比) 和 Head Pose (Yaw/Pitch) 数据。
